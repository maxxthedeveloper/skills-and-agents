# Work Page Layout System Reference

## Layout Types

| Layout | Outer Aspect | Inner Mockup | Grid Span | Use For |
|--------|-------------|--------------|-----------|---------|
| `desktop-contained` | 3:2 | 16:10 | 2 cols | Desktop app/website screenshots in a frame |
| `desktop-full` | 3:2 | full-bleed | 2 cols | Wide photos, full-width images |
| `mobile-contained` | 2:3 | 9:19.5 | 1 col | Phone app screenshots in a device frame |
| `portrait-full` | 2:3 | full-bleed | 1 col | Tall portrait photos or images |
| `component` | 1:1 | full-bleed | 1 col | Square UI components, logos, icons, detail shots |

## Grid Behavior

- The grid is 2 columns
- 2-col layouts (`desktop-contained`, `desktop-full`) span the full width
- 1-col layouts (`mobile-contained`, `portrait-full`, `component`) sit side by side in pairs
- Mixing 1-col types in a row is fine (e.g., `component` + `component`, or `mobile-contained` + `portrait-full`)

## WorkItem Interface

```typescript
export type TileLayout =
  | "desktop-contained"
  | "desktop-full"
  | "mobile-contained"
  | "portrait-full"
  | "component"

export interface WorkItem {
  id: string                    // Sequential string: "1", "2", "3"...
  projectName: string           // Display name ‚Äî empty string "" means no hover state
  layout: TileLayout            // One of the 5 layout types
  imageSrc?: string             // Path relative to public/: "/work/filename.jpg"
  videoSrc?: string             // Path to .mp4 video: "/work/filename.mp4"
  projectType?: string          // Subtitle tag: "Waitlist Page", "UI Kit", etc.
  builtWith?: {                 // Collaborator credit (optional)
    name: string                // Display name
    avatarSrc?: string          // Avatar image path (optional)
    url?: string                // Link to collaborator profile
    label?: string              // Custom label (defaults to "Built by")
  }
  innerRadius?: number          // Custom inner border radius (overrides DevPanel default of 20px)
  hoverBarTheme?: "light" | "dark"  // ONLY for full-bleed layouts, never on contained
}
```

## Video Tiles

Video tiles require BOTH `imageSrc` (poster image) and `videoSrc` (.mp4 path). The poster image shows immediately while the video lazy-loads.

**How TileMedia works:**
- IntersectionObserver lazy-loads video when tile is within 400px of viewport
- Videos play/pause based on viewport visibility (20% threshold)
- `getVideoSources()` automatically derives `.webm` + `.mp4` paths from `videoSrc`
- `getImageSources()` automatically derives `.avif` + `.webp` + fallback from `imageSrc`
- When `prefers-reduced-motion` is enabled, only the poster image is shown (no video)

**Required assets for a video tile:**
- `.mp4` file (the video source)
- `.webm` file (VP9 variant ‚Äî generated via `ffmpeg -c:v libvpx-vp9 -crf 35 -b:v 0 -an`)
- Poster `.jpg` (extracted from video via `ffmpeg -ss 0.5 -frames:v 1 -q:v 2` or user-provided)
- Poster `.avif` + `.webp` (generated by `scripts/optimize-images.mjs`)

## Hover State

The hover info bar renders only when `projectName` is truthy OR `builtWith` is present. To create a tile with no hover state, set `projectName: ""` and omit `builtWith`.

**Theme rules:**
- Full-bleed layouts (`desktop-full`, `portrait-full`, `component`): set `hoverBarTheme` based on image brightness ‚Äî `"light"` for dark images, `"dark"` for light images
- Contained layouts (`desktop-contained`, `mobile-contained`): NEVER set `hoverBarTheme` ‚Äî the hover bar sits on the outer background where default dark text is always correct

## File Paths

- Data model: `app/work/data.ts`
- Tile component: `app/components/work/Tile.tsx`
- Grid page: `app/work/page.tsx`
- Blur data: `app/work/blur-data.ts` (auto-generated by `scripts/generate-lqip.mjs`)
- Assets directory: `public/work/`
- Image src format: `/work/filename.jpg` (no `public/` prefix)

## Asset Naming

Images in `public/work/` can use any name. Common patterns:
- `desktop-screenshot project-name.jpg` ‚Äî for desktop contained
- `desktop-full-bleed description.jpg` ‚Äî for desktop full
- `mobile-screenshot project-name.jpg` ‚Äî for mobile contained
- `project-component.png` ‚Äî for component tiles (use .png if transparency needed)
- `project-name.mp4` + `project-name.webm` ‚Äî video file pair
- `project-name-poster.jpg` ‚Äî poster frame extracted from video

Each source image gets auto-generated variants via `scripts/optimize-images.mjs`:
- `filename.avif` ‚Äî AVIF variant (quality 65)
- `filename.webp` ‚Äî WebP variant (quality 75)

## Design Tokens

- Outer container: `bg-transparent-4` background, squircle corners via `corner-smoothing` library
- Inner mockup: `bg-primary` background with squircle corners
- Corner smoothing: `cornerSmoothing={0.6}`
- Both border radii controlled by DevPanel context (defaults: outer 0px, inner 20px)
- Container padding: 12% (from DevPanel defaults)
